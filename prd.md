## 摘要

> 下游 Agent 请优先阅读本节，需要细节时再查阅完整文档。

- **核心目标**：为独立调香师 Lina 打造高端个人作品集网站，通过沉浸式视觉体验传递"东方草本"品牌气质，实现作品展示与预约咨询
- **目标用户**：潜在客户、品牌合作方、媒体编辑、调香爱好者
- **关键功能**：朱砂烟雾 Hero 动画、作品卡片展示、调香流程时间轴、极简预约表单、墨迹自定义光标
- **技术约束**：纯前端（React 19 + TypeScript + Three.js + GSAP），无后端，表单依赖 formspree，LCP < 2.5s
- **优先级**：全量交付（MVP = 首屏 + 作品展示 + 预约表单，其余为应有功能）

## 需求穿透分析

### 需求分层模型

#### 1. 显性需求（用户明确表达的）
- 全屏朱砂红烟雾动画 Hero，Three.js Fragment Shader 驱动，8秒循环
- 五个页面区块：Hero / About / Works / Process / Contact
- 色板：墨黑 #0A0A0A / 暖米白 #F5F0E8 / 朱砂红 #C9302C
- 墨迹自定义光标，移动拖出晕染轨迹，hover 圆点放大
- GSAP ScrollTrigger 滚动动画
- clip-path 从左向右揭幕动画
- 响应式，移动端优雅降级
- 纯前端，表单用 formspree
- LCP < 2.5s

#### 2. 隐性需求（用户想到但未表达的）
- **品牌一致性**：Lina 在京都学习调香 8 年，"东方草本"主题贯穿全站，需要日式侘寂美学融入设计细节
- **作品展示深度**：每张香水作品卡片需要支持多图（瓶身、调香笔记、成分图谱）
- **移动端手势**：横向滚动时间轴在移动端需支持触摸滑动，且提供替代的垂直布局
- **页面加载体验**：Three.js Shader 资源较重，需要 loading 过渡动画而非白屏等待
- **SEO 基础**：作品集网站需要被搜索引擎收录（meta 标签、Open Graph、语义化 HTML）
- **无障碍基础**：自定义光标需支持 prefers-reduced-motion，Shader 动画可关闭
- **性能降级策略**：不支持 WebGL 的设备需有优雅降级（静态图片或简化 CSS 动画）

#### 3. 潜在需求（用户尚未意识到的）
- **品牌叙事线**：从 Hero 的抽象烟雾 → About 的个人故事 → Works 的作品呈现 → Process 的匠人精神 → Contact 的行动号召，形成完整的情感漏斗
- **社交媒体传播**：每个作品页应有独立的 Open Graph 卡片，方便分享时被媒体引用
- **数据埋点**：需要基础分析（访问路径、作品停留时长、表单提交转化），为后续运营提供依据
- **字体 fallback 策略**：Canela/Editorial New 为商业字体，需确认授权或准备 fallback 链
- **CDN 资源分发**：Three.js + 大图片资源，需考虑全球加载速度
- **内容可维护性**：作品数据应从 JSON/配置文件读取，方便后续自行增删

#### 4. 惊喜需求（超出预期的创新点）
- **气味可视化**：在作品卡片 hover 时，以粒子动画"绽放"出该香水的前中后调成分，让"看不见的香气"被看见
- **暗色沉浸模式**：全站默认暗色，但在 About 区切换为暖米白底色，营造"从黑夜步入工作室"的情绪转换
- **滚动进度指示器**：以"香"字形 SVG 作为滚动进度条，随滚动进度逐渐被"烟雾填充"
- **声音氛围**：可选的环境音（白噪音 + 轻微水滴声），增强沉浸感，默认关闭
- **预约表单仪式感**：提交后显示一段调香箴言 + 烟雾消散动画，让"提交"本身成为品牌体验

## 1. 概述

### 1.1 背景
Lina 是一位在京都学习调香 8 年的独立调香师，作品以"东方草本"为主题。目前她缺乏一个能充分传递品牌气质、展示作品的线上平台。现有社交媒体展示碎片化，无法承载"高端调香师"的品牌定位。需要一个纯前端作品集网站，兼具视觉震撼力与极简的预约功能，作为她在数字世界的"第一张名片"。

### 1.2 目标
1. **品牌传递**：通过沉浸式视觉体验（朱砂烟雾、墨迹光标、日式排版），让访客在进入网站 3 秒内感知 Lina 的品牌气质
2. **作品展示**：清晰、优雅地展示香水作品，每件作品配有成分、故事、调香笔记
3. **转化驱动**：通过极简预约表单，引导潜在客户和品牌合作方发起咨询
4. **性能标杆**：在动画丰富的前提下保持 LCP < 2.5s，达到 Awwwards 获奖作品水准

### 1.3 成功指标
- [ ] LCP < 2.5s（移动端 + 桌面端）
- [ ] 首屏加载时间 < 3s（3G 网络模拟下 < 5s）
- [ ] Hero 动画帧率稳定 60fps（桌面端）/ 30fps（移动端）
- [ ] 页面可用性评分 Lighthouse Performance > 85
- [ ] 预约表单提交转化率 > 3%（上线 3 个月后追踪）
- [ ] 移动端布局无水平滚动、无内容溢出

## 3. 功能需求

### FR-001：朱砂烟雾 Hero 首屏
- **描述**：当用户访问网站时，全屏展示朱砂红色烟雾动画，使用 Three.js + Fragment Shader 实现 volumetric smoke 效果，纯黑底色 #0A0A0A，动画 8 秒循环，右下角有橙红光晕，叠加胶片颗粒感滤镜
- **优先级**：必须有
- **依赖**：Three.js、@react-three/fiber 库加载
- **验收标准**：
  - [ ] AC-1：Hero 区域占满视口（100vh），背景色 #0A0A0A
  - [ ] AC-2：Fragment Shader 实现烟雾 volumetric 效果，颜色从 #C9302C 渐变至 #8B1A1A
  - [ ] AC-3：烟雾动画无缝循环，周期 8 秒
  - [ ] AC-4：右下角有橙红（#FF6B3A 附近色值）光晕效果，透明度随烟雾变化
  - [ ] AC-5：胶片颗粒感滤镜叠加在烟雾之上（post-processing pass）
  - [ ] AC-6：桌面端帧率稳定 ≥ 60fps，移动端 ≥ 30fps
  - [ ] AC-7：不支持 WebGL 的设备降级为静态烟雾图片（预渲染 PNG）
  - [ ] AC-8：prefers-reduced-motion 用户降级为静态图片或简化 CSS 渐变
  - [ ] AC-9：首屏标题文字在烟雾之上清晰可读，使用 Instrument Serif 字体

### FR-002：About 区 — 个人故事
- **描述**：左文右图布局，左侧为 Lina 的调香故事（京都 8 年学习经历、"东方草本"理念），右侧为黑白工作室照片，从 Hero 的暗色过渡到暖米白 #F5F0E8 底色
- **优先级**：应该有
- **依赖**：黑白工作室照片素材
- **验收标准**：
  - [ ] AC-1：布局为左侧文字（60%）+ 右侧图片（40%），桌面端
  - [ ] AC-2：底色从 #0A0A0A 平滑过渡至 #F5F0E8（约 200px 渐变过渡区）
  - [ ] AC-3：文字使用 Instrument Serif（标题）+ 思源宋体（中文正文）
  - [ ] AC-4：工作室照片为黑白处理，右侧边缘有柔和羽化
  - [ ] AC-5：滚动进入时，文字逐行淡入（GSAP ScrollTrigger 驱动）
  - [ ] AC-6：移动端改为上图下文垂直布局

### FR-003：Works 区 — 香水作品展示
- **描述**：展示 Lina 的香水作品，以卡片形式呈现，每张卡片包含作品名称、香调图谱、简短描述。进入视口时使用 clip-path 从左向右揭幕动画
- **优先级**：必须有
- **依赖**：作品图片、名称、描述数据（JSON 配置）
- **验收标准**：
  - [ ] AC-1：卡片网格布局，桌面端 2-3 列，平板端 2 列，移动端 1 列
  - [ ] AC-2：每张卡片进入视口时，使用 clip-path: inset() 从左向右展开，动画时长 600ms，easing: cubic-bezier(0.25, 0.46, 0.45, 0.94)
  - [ ] AC-3：卡片 hover 时显示香调信息（前调/中调/后调）
  - [ ] AC-4：卡片包含：作品名称（Instrument Serif）、主图、一句描述、香调标签
  - [ ] AC-5：作品数据从 JSON 文件读取，方便后续增删
  - [ ] AC-6：移动端卡片全宽，滑动切换
  - [ ] AC-7：每张作品卡片可点击展开详情（Modal 或独立路由）

### FR-004：Process 区 — 调香流程时间轴
- **描述**：横向滚动的调香流程时间轴，展示从灵感采集到成品的完整过程，每个节点有图标、标题、简短说明
- **优先级**：应该有
- **依赖**：流程步骤数据（至少 5 个节点）
- **验收标准**：
  - [ ] AC-1：桌面端为横向滚动容器，支持鼠标拖拽滚动
  - [ ] AC-2：移动端改为垂直堆叠布局，保持相同视觉风格
  - [ ] AC-3：每个节点包含：序号、图标、标题、1-2 句说明
  - [ ] AC-4：节点之间以细线连接，线随滚动进度逐渐显现
  - [ ] AC-5：当前视口中心的节点高亮显示（#C9302C 色）
  - [ ] AC-6：滚动进入时节点依次淡入，间距均匀
  - [ ] AC-7：至少包含 5 个流程节点（灵感采集 → 原料甄选 → 配方调试 → 陈化成熟 → 成品发布）

### FR-005：Contact 区 — 预约表单
- **描述**：极简风格的预约表单，包含姓名、邮箱、咨询类型、留言字段，提交至 formspree
- **优先级**：必须有
- **依赖**：formspree 端点配置
- **验收标准**：
  - [ ] AC-1：表单字段：姓名（必填）、邮箱（必填+格式校验）、咨询类型（下拉：个人定制/品牌合作/媒体采访/其他）、留言（选填）
  - [ ] AC-2：提交按钮使用朱砂红 #C9302C，hover 时变为深红 #8B1A1A
  - [ ] AC-3：提交成功后显示烟雾消散动画 + 调香箴言
  - [ ] AC-4：表单验证错误以行内提示显示，不使用 alert
  - [ ] AC-5：邮箱格式校验使用正则 /^[^\s@]+@[^\s@]+\.[^\s@]+$/
  - [ ] AC-6：提交过程显示 loading 状态（按钮文字变为"发送中..."）
  - [ ] AC-7：formspree 提交失败时显示友好错误提示，允许重试
  - [ ] AC-8：表单区域底色回归 #0A0A0A，形成首尾呼应

### FR-006：墨迹自定义光标
- **描述**：全站自定义光标，默认为一个墨点，移动时拖出晕染墨迹轨迹，hover 可交互元素时圆点放大
- **优先级**：应该有
- **依赖**：cursor CSS 隐藏、Canvas/Div 光标层
- **验收标准**：
  - [ ] AC-1：隐藏系统默认光标（cursor: none），替换为自定义墨点光标
  - [ ] AC-2：墨点光标为直径 12px 的圆形，颜色 #0A0A0A（暗色区）/ #C9302C（亮色区）
  - [ ] AC-3：移动时拖出渐隐的墨迹轨迹，轨迹粒子逐渐缩小至消失，持续时间 800ms
  - [ ] AC-4：hover 可交互元素（链接、按钮、卡片）时，墨点放大至 48px，半透明，显示元素类型提示
  - [ ] AC-5：移动端自动禁用自定义光标（使用系统默认触摸交互）
  - [ ] AC-6：prefers-reduced-motion 用户降级为静态光标（不拖轨迹）
  - [ ] AC-7：光标层 z-index 最高，不阻挡下方元素交互（pointer-events: none）

### FR-007：全局导航
- **描述**：固定在页面顶部的极简导航栏，包含 Logo（Lina 手写签名风格）和锚点链接（About / Works / Process / Contact），滚动时自动隐藏/显示
- **优先级**：必须有
- **依赖**：GSAP ScrollTrigger 或 IntersectionObserver
- **验收标准**：
  - [ ] AC-1：导航栏固定于顶部，背景透明，文字颜色 #F5F0E8
  - [ ] AC-2：向下滚动 100px 后导航栏淡出，向上滚动时淡入
  - [ ] AC-3：点击锚点链接平滑滚动至对应区域（smooth scroll）
  - [ ] AC-4：移动端导航折叠为汉堡菜单，展开后全屏遮罩
  - [ ] AC-5：当前所在区域的导航项高亮显示

### FR-008：页面加载过渡
- **描述**：首屏加载时显示品牌加载动画，Three.js 资源就绪后平滑过渡到 Hero
- **优先级**：应该有
- **依赖**：资源预加载策略
- **验收标准**：
  - [ ] AC-1：加载页面为纯黑 #0A0A0A 背景，中央显示 Lina 签名淡入
  - [ ] AC-2：Three.js Shader 初始化完成后，加载页淡出（300ms）
  - [ ] AC-3：若 3 秒内资源未就绪，跳过加载动画直接显示内容
  - [ ] AC-4：加载动画不超过 3 秒（超时自动跳过）

## 4. 非功能需求

### NFR-001：性能要求
- **类型**：性能
- **描述**：在动画丰富的前提下保证核心性能指标
- **指标**：
  - LCP（Largest Contentful Paint）< 2.5s
  - FID（First Input Delay）< 100ms
  - CLS（Cumulative Layout Shift）< 0.1
  - 首屏 JavaScript 体积 < 200KB（gzip 后）
  - Hero 动画帧率：桌面端 ≥ 60fps，移动端 ≥ 30fps
  - 图片使用 WebP 格式，Hero 降级图片 < 100KB

### NFR-002：响应式适配
- **类型**：兼容性
- **描述**：在桌面端、平板、移动端均有良好体验
- **指标**：
  - 支持断点：≥ 1200px（桌面）、768-1199px（平板）、< 768px（移动）
  - 移动端不使用 hover 效果（改为点击触发）
  - 横向滚动时间轴在移动端转为垂直布局
  - 自定义光标在移动端自动禁用
  - 无水平滚动条溢出

### NFR-003：无障碍访问
- **类型**：无障碍
- **描述**：尊重用户的系统偏好，提供可访问的降级体验
- **指标**：
  - 支持 prefers-reduced-motion：禁用所有动画，替换为静态展示
  - 所有图片提供 alt 文本
  - 表单字段关联 label 元素
  - 键盘导航可达所有交互元素
  - 色彩对比度满足 WCAG AA 标准（≥ 4.5:1）

### NFR-004：SEO 基础
- **类型**：搜索引擎优化
- **描述**：确保网站可被搜索引擎正确索引和展示
- **指标**：
  - 语义化 HTML 结构（header / main / section / footer）
  - 每个页面区域有合理的 heading 层级（h1-h3）
  - Open Graph meta 标签（title / description / image）
  - 站点 title 格式："Lina — 东方草本调香师"
  - meta description 包含关键词：独立调香师、东方草本、京都、香水定制

### NFR-005：安全与隐私
- **类型**：安全
- **描述**：保护用户提交的表单数据，遵守隐私规范
- **指标**：
  - formspree 端点使用 HTTPS
  - 不在前端代码中暴露 formspree 端点 URL（使用环境变量）
  - 表单提交不收集 IP 地址等额外信息
  - 隐私政策链接（页脚），说明数据用途

### NFR-006：可维护性
- **类型**：可维护性
- **描述**：方便 Lina 后续自行更新作品内容和文字
- **指标**：
  - 所有作品数据存储在独立的 JSON 配置文件
  - 文本内容（故事描述、流程说明）可通过配置文件修改
  - 图片资源按模块组织（hero / about / works）
  - 代码使用 TypeScript 类型定义，组件文档完整

## 5. 用户故事

### US-001：浏览首页留下第一印象
- **作为** 品牌采购方
- **我想要** 在打开网站的前 3 秒内看到震撼的视觉效果
- **以便** 快速判断 Lina 的品牌调性是否与我方匹配

## 7. 风险与依赖

### 7.1 风险
| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| Three.js Shader 在低端设备卡顿 | 中 | 高 | WebGL 能力检测 + 降级静态图片；移动端降低 Shader 分辨率 |
| Fragment Shader 包体积过大影响 LCP | 中 | 高 | 代码分割，Hero Shader 优先加载；使用 CDN；预编译 shader 为 .glsl 文件 |
| 商业字体（Canela/Editorial New）授权问题 | 低 | 中 | 优先使用 Instrument Serif（Google Fonts 免费）+ 思源宋体；Canela 仅作为 fallback 声明 |
| formspree 服务不稳定 | 低 | 中 | 表单提交失败时本地缓存数据 + 提示用户稍后重试；提供备选联系方式（邮箱直链） |
| GSAP ScrollTrigger 与 Three.js Canvas 渲染冲突 | 低 | 低 | 使用 React 严格模式隔离；GSAP 动画仅操作 DOM，Canvas 使用独立的 rAF 循环 |
| 墨迹光标在高频移动时性能下降 | 中 | 中 | 限制粒子生成频率（每 16ms 最多 1 个）；使用 CSS transform 而非 JS 动画 |
| 移动端 Safari 对 clip-path 兼容性 | 低 | 低 | 测试 Safari iOS clip-path 支持，必要时使用 opacity + transform 替代方案 |
| 胶片颗粒感 post-processing 影响帧率 | 中 | 低 | 移动端关闭 post-processing 或使用简化版（CSS noise 纹理） |
